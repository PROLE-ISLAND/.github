# セキュリティポリシー

## 脆弱性の報告

セキュリティ上の問題を発見した場合、**公開Issueではなく**以下の方法で報告してください。

### 報告方法

1. GitHub Security Advisory を使用
   - リポジトリの「Security」タブ → 「Report a vulnerability」

2. または直接連絡
   - 組織オーナーに直接DM

### 報告に含める情報

- 脆弱性の種類（XSS、SQLi、認証バイパス等）
- 再現手順
- 影響範囲の評価
- 可能であれば修正案

### 対応プロセス

1. 報告受領の確認（24時間以内）
2. 初期評価（72時間以内）
3. 修正パッチ作成
4. セキュリティアドバイザリ公開
5. 報告者への謝辞

## サポートバージョン

| バージョン | サポート状況 |
|-----------|-------------|
| 最新リリース | ✅ 完全サポート |
| 1つ前のメジャー | ⚠️ セキュリティ修正のみ |
| それ以前 | ❌ サポート終了 |

## セキュリティベストプラクティス

- 機密情報は環境変数で管理（.envはコミットしない）
- 依存関係は定期的に更新（Dependabot有効）
- RLSでデータアクセス制御
